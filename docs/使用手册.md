## 一、完整安装（约 30 分钟）

不需要懂 Python，每一步都告诉你「怎么确认这步成功了」。

### 你需要准备

| 东西 | 说明 |
|---|---|
| **两个 QQ 号** | 一个当机器人（小号），一个是你自己。**机器人号会被自动化操作，有被封的风险，别用主号。** |
| **一个 QQ 群** | 用来接收推送。机器人和你都要在里面 |
| **要监听的羊毛群** | 机器人必须已经在群里（不用是管理员） |
| **一台一直开着的电脑** | 关机了就不推了 |
| 一个 AI 模型的 API Key | 可选但强烈建议。DeepSeek 一个月大概几毛钱；也支持 Kimi / 智谱 / 通义 / OpenAI |
| 微博 Cookie | 只有你要监控微博博主时才需要 |

### 1. 装 Python

到 https://www.python.org/downloads/ 下载 **3.10 或更新**的版本。

安装时**务必勾选** `Add Python to PATH`（安装界面最下面那个复选框）。漏了这一步，后面所有命令都会报「不是内部或外部命令」。

确认成功：按 `Win + R` 输入 `cmd` 回车，在黑框里敲 `python --version`，看到 `Python 3.10.x` 之类就对了。

> 提示「不是内部或外部命令」= 没勾 PATH。重新运行安装包 → Modify → 一路 Next → 勾上 `Add Python to environment variables`。

### 2. 下载本项目

**不用装 git，直接下压缩包** → **[点这里下载最新版 zip](https://github.com/irito045/wool-hunter/releases/latest)**（在 Assets 里选 `wool-hunter-v*.zip`，约 250KB）

解压到任意位置（比如 `D:\wool-hunter`）。然后在这个文件夹里按住 `Shift` + 右键 → 「在此处打开 PowerShell 窗口」，装依赖：

```bash
pip install -r requirements.txt
```

<details>
<summary>会用 git 的话，也可以 clone</summary>

```bash
git clone https://github.com/irito045/wool-hunter.git
cd wool-hunter
pip install -r requirements.txt
```
</details>

> 电脑装了多个 Python 时，`pip` 可能装到了别的版本里。保险写法是 `python -m pip install -r requirements.txt`。
>
> GitHub 下载慢的话，多刷几次，或者用浏览器的下载工具续传——文件只有 250KB 左右，一般几秒就好。

### 3. 下载 NapCatQQ（让机器人能收发 QQ 消息）

本项目自己**不能**登录 QQ，它需要 [NapCatQQ](https://github.com/NapNeko/NapCatQQ) 当中间人。

去 [Releases](https://github.com/NapNeko/NapCatQQ/releases) 下载最新版，Windows 选 **`NapCat.Shell.Windows.OneKey`**（自带 QQ，不用装 QQ，也不用管理员权限），解压到任意位置。

**就这样，别去启动它。**下一步控制台会替你做完剩下的事：无窗口启动、扫码登录、自动写好反向 WebSocket 配置。你不需要打开 NapCat 的黑框，也不需要进它的 WebUI 手点。

> **只有 OneKey 版能被控制台接管。** 非 OneKey 的 Shell 版要读注册表找系统装的 QQ、还要管理员权限，控制台管不了它——但也**不会因此报错**：你自己启动 NapCat，只要它连上了 bot，「运行」页照样是绿的，其余功能一个不少。

### 4. 填配置

**双击项目里的 `console.bat`**，打开桌面控制台，切到「配置」页。

不用复制文件、不用开记事本、不用记变量名。三项必填，每一项下面都写着它是干什么的。填完点「保存配置」，`.env` 会自动生成。

```
必填
  监听哪些群   [                ]   ← bot 要盯着看的羊毛群
  推送到哪些群 [                ]   ← 找到好价往哪儿发
  管理员 QQ号  [                ]   ← 谁能用管理命令
```

填错了它会拦住你，并用人话告诉你错在哪。

> 控制台**不需要 bot 在跑**就能打开。这正是它存在的意义——你现在还没启动过 bot。

**群号怎么看？** 电脑版 QQ：打开群 → 右上角「群设置」。手机版：群聊 → 右上角三条杠。

> ⚠️ `推送到哪些群` 有两个作用：既是推送目标，也是 `/w` 命令唯一生效的群。而且在群里用命令**必须 @ 机器人**——详见[第五节](#五在群里怎么用命令要--机器人)。

**AI 模型 API Key**（可选，建议填）：

「配置」页的 AI 分区有一个**服务商下拉**——支持 DeepSeek、Kimi、智谱 GLM、通义千问、OpenAI，选一家会自动填好接口地址和模型名，你只要把对应平台申请到的 key 填进去。用别家兼容 OpenAI 接口的服务就选「自定义」，自己填地址和模型。

以 DeepSeek 为例：到 https://platform.deepseek.com/ 注册 → 「API keys」创建 → 最低充 1 块钱能跑很久（每条消息约 0.00002 元）→ 填进控制台，点旁边的**「测试」**确认能用（测试会拿你选的地址和模型真打一次接口）。

不填会怎样？AI 质量把关会**全部放行**，只剩正则规则挡噪音。能跑，但签到打卡、拉人头那类帖子会漏进来一些。

### 5. 登录 QQ

切到「运行」页，找到 **NapCat** 那张卡片，点 **「扫码登录」**。

会弹出一个窗口显示二维码，用**机器人那个小号**的手机 QQ 扫。扫完控制台会：

1. 记住这个 QQ 号（下次点「启动」就免扫码了）
2. 自动把反向 WebSocket 写进 NapCat 的配置
3. 重启 NapCat 让配置生效

全程不弹黑框。如果二维码显示不出来，弹窗里还有一个可以复制到手机上打开的链接。

> 如果控制台找不到 NapCat，去「配置」页填一下「NapCat 安装目录」（含 `NapCatWinBootMain.exe` 的那个文件夹）。

### 6. 启动 bot

还是「运行」页，先看**环境体检**。六项都是真去验的，不是看你填没填：

```
✔ Python 版本   3.12.4
✔ 依赖包        都装好了                  ← 缺了会有「一键安装」按钮
✔ 配置文件      必填项齐了
✔ NapCat        已登录，并且连上 bot 了
✔ AI 模型       可用（deepseek-chat）
! 微博          没配博主 UID，只监听 QQ 群
```

> NapCat 这一项**不是**看端口通不通。它在「等待扫码」的时候端口就已经开了——只有真正登录、并且和 bot 建立了连接，这里才会是绿的。

前四项绿了就点右上角的 **「启动」**。下面的实时日志里应该出现：

```
[INFO] uvicorn.error: Uvicorn running on http://127.0.0.1:8081
[INFO] internal_api: 🔌 内部接口已挂载：POST /api/internal/resend（仅本机 + token）
[INFO] uvicorn.error: "WebSocket /onebot/v11/ws" [accepted]    ← 这行代表 NapCat 连上了！
```

**最后那行是关键。** 没有它 = NapCat 没连上，机器人收不到任何消息。

### 7. 验证它真的在工作

1. 在**私聊**里给机器人发 `/w`，应该回一大段帮助菜单。没反应 → NapCat 没连上，回看第 5 步。
2. 发 `/w low 20`（订阅：到手价 ≤20 元就推）
3. 等羊毛群里出现一条便宜货。等不及的话去控制台「总览」页，那里实时显示每条消息**为什么推**或**为什么被拦**；双击一行还能告诉它「这条判错了」。

## 二、桌面控制台

双击 `console.bat`，这是操作这个项目的**唯一界面**。不弹黑框（用 `pythonw` 起窗口程序），它自己就是看门狗——bot 崩了 3 秒后自动拉起来。

**bot 没跑的时候也能用**——首次部署、bot 崩了、配置填错了，恰恰都是一个寄生在 bot 进程里的界面打不开的时候。

| 页 | 干什么 |
|---|---|
| 总览 | 统计、最近判定、点一条反馈「判定准不准」、补发、暂停推送、屏蔽词 |
| 运行 | 环境体检（逐项**真去验**）、**启停 NapCat 和扫码登录**、彩色实时日志 |
| 日志 | 全屏实时日志（和「运行」页同源，但更大一屏） |
| 配置 | 表单填 `.env`，不用开文本编辑器。密钥只显示掩码；微博 Cookie 可以点「扫码登录」自动获取 |
| 订阅 | 看和改所有人的订阅（三类合一） |
| 品类 | 品类 → 关键词表 |
| 拦截 | 10 类噪音的开关 |

零依赖（tkinter 是 Python 自带的）。除了「补发」，所有页面都直接调 `services/` 里 bot 自己用的函数——不走 HTTP，任何逻辑都没有第二份。

「补发」是唯一的例外：控制台是独立进程，发不了 QQ 消息，所以它通过 bot 上一个**只监听本机、带随机 token** 的内部接口（`POST /api/internal/resend`）来做。

**`.env` 的改动要点「重启」才生效**（配置页顶部有横幅提醒），其余（订阅 / 品类 / 拦截 / 暂停）都是热加载、改完立刻生效。

### 最小化到托盘

点右上角的 **✕ 是缩到系统托盘**，不是退出——bot 照常在后台推送，也不占任务栏。

- **双击**托盘图标：把控制台叫回来
- **右键**托盘图标：显示控制台 / 启动·停止 bot / 退出控制台

想**真正退出**，右键托盘 →「退出控制台」。这时它会列出还在后台跑的东西（bot、NapCat），问你要不要一起停掉；选「否」它们就继续跑，推送不中断。**NapCat 只会停掉它自己安装目录下的进程**，不会碰你正在用的 QQ。

`console.bat` 是**唯一**的入口。调试时可以 `python bot.py` 在前台跑一次，但别和控制台同时开——两个 bot 实例会抢 8081 端口和 NapCat 的连接，表现是「消息时有时无」。

## 三、三类订阅 & 价格上限

| 类型 | 命令 | 命中条件 |
|---|---|---|
| **低价** | `/w low 20` | 估算到手价 ≤ 20 元。**纯门槛，不看是什么商品** |
| **关键词** | `/w add 耳机`<br>`/w add 显示器 ktc` | 词命中。单词会走 AI 语义扩展（订「抽纸」也收到纸巾/手帕纸）；多词是字面 AND |
| **品类** | `/w cat 零食` | 属于该品类（词表匹配，词表没收录的由 AI 兜底归类） |

三类可以同时订，命中任一即推送，每条消息对同一个目标最多推一次。

「在哪里发的命令，就推送到哪里」：群里发 → 推到该群；私聊发 → 私信你。**群订阅属于群**，同群任何人都能查看、增删。

### 价格上限：总价，还是单价？

关键词和品类订阅可以**再附加一个价格上限**，表达「零食 且 ≤20 元」这种最常见的需求。而上限本身有两种算法：

| 口径 | 意思 | 例子 |
|---|---|---|
| **总价**（默认） | 你实际掏多少钱 | 「券后 19.9」→ 19.9 元 |
| **单价** | 每一件/瓶/盒多少钱 | 「拍 12 件，折 1.4 元/件」→ 1.4 元 |

```
/w low 20              到手价 ≤20 元
/w low 单价 2          每瓶/每件 ≤2 元（整箱矿泉水这种）
/w cat 零食 ≤20        零食，且到手价 ≤20 元
/w add 矿泉水 单价≤2   出现「矿泉水」，且每瓶 ≤2 元
/w add 耳机            不看价，全推
```

**为什么要分两种？** 「拍 12 件，折 1.4 元/件」这条消息里，程序**不会替你做乘法**——它不知道你是不是真要拍 12 件，也不知道有没有别的凑单方式。所以它老老实实地说：这条读不出总价（真实是 16.8 元），但读得出单价 1.4 元。订「总价≤20元」的人收不到它，订「单价≤2元」的人收得到。

单价只在**能数着买的单位**上成立：件 / 瓶 / 盒 / 包 / 袋 / 支 / 管 / 桶 / 听 …。像「0.014 元/抽」的抽纸、「0.008 元/ml」的牛奶，那是规格单价，不是你能买一份的价格，程序不会把它当单价——否则「单价≤1元」会命中市面上每一包纸。

设了上限之后，**对应口径估不出价格的帖子不会推**。既然你明说了要 ≤N 元，一个价都读不出来的帖子就不该塞给你。

## 四、消息怎么被过滤

```
消息进来
  ├─ 跨源去重（30 分钟内近似重复 → 丢弃）
  ├─ 噪音拦截（10 类可开关：外卖红包券 / 银行支付理财 / 签到打卡任务 / 打车券 …）
  ├─ 垃圾帖过滤（只有链接和数字、没有商品名）
  ├─ AI 质量把关：这是「具体商品 + 可购买的优惠」吗？（不判价格，拿不准就放行）
  └─ 匹配三类订阅 → 推送
```

10 类噪音拦截可以在控制台「拦截」页里**逐类开关**。关掉某类 = 你想收这类，它会直接放行。默认全开。

## 五、在群里怎么用（命令要 @ 机器人）

为了**隐私**，机器人在用户群里**只回应「点名找它」的消息**，不对群友日常闲聊做任何反应。

- **群里**：用 `/w`、`/help`、`/查` 这些命令，都要 **@ 一下机器人**（或引用它发过的消息再打命令）。没 @ 它，它就当没看见——哪怕你消息里恰好带了 `/w`。
- **私聊**：不受影响，直接发命令就行。
- **羊毛群**：它照常读群里每一条消息来找好价，这个不受影响。同一个群若既是羊毛群又是用户群，好价照抓，只是你发命令要 @。

```
群里：  @羊毛猎人 /w low 20      ✅ 会应答
群里：  /w low 20               ❌ 没 @，装作没看见
私聊：  /w low 20               ✅ 照常应答
```

> 引用回复报「贵了」这类反馈是例外：引用机器人自己发的那条消息，本身就算「点名找它」，不用再手动 @。

## 六、命令一览

```
/w low 20              低价订阅：到手价≤20元就推   /w low off 退订
/w low 单价 2          低价订阅：每件/瓶/盒≤2元
/w add 耳机            关键词订阅（多词：/w add 显示器 ktc）
/w add 耳机 ≤50        关键词 + 总价上限
/w add 矿泉水 单价≤2   关键词 + 单价上限
/w cat 零食            品类订阅          /w cat 看所有品类
/w cat 零食 ≤20        品类 + 价格上限
/w cat addword 零食 溶豆   共享品类表，谁都能改
/w list / del / on / off   查看 / 删除 / 暂停 / 恢复订阅
/w block add 代购      屏蔽词（按群 / 按私聊分作用域）
/查 纸巾               翻最近两天含该词的羊毛（含被拦截的）

管理员（仅私聊）：/w list all · /w pause · /w resume · /w log · /w reload · /w weibo
                  /w broadcast <消息>   群发给所有订阅者
```

> 群里所有命令都要 **@ 机器人**，私聊直接发。见[第五节](#五在群里怎么用命令要--机器人)。

**`/w broadcast`（群发）分两步，防手滑：**
1. `/w broadcast 本工具今晚10点维护半小时` → 机器人回你「即将发给 私聊 N 人 ＋ 群 M 个」+ 内容预览
2. 看没问题，直接回 **`/w go`** → 才真正群发（5 分钟内有效，过期作废）

**原样发送**——你发什么内容、什么格式，就照样发出去，不加任何抬头或前缀。群发不受「暂停推送」影响（维护通知就得在暂停时也发得出去），也不会写进统计。目标是**所有订阅者去重**：私聊订阅的人 + 有群订阅的群（群必须在推送白名单里）。

## 七、反馈

引用（回复）任意一条推送，然后发一个词：

| 回复 | 效果 |
|---|---|
| `不想要` / `不要这个` / `跳过` / 👎 | AI 提取商品核心词，自动加进屏蔽词 |
| `不是羊毛` | 同上，并记一票 |
| `贵了` / `太贵` / `不值` | 只记一票，**不屏蔽**（用于复盘到手价估算准不准） |
| `好价` / `划算` / 👍 | 正面反馈 |

群里必须**引用**才算反馈，否则群友闲聊说句「不错」会被误当反馈。也正因为是引用机器人发的消息，它自然满足「点名找它」，不用额外 @。

## 八、微博监控（可选）

除了 QQ 群，机器人也能盯着微博博主发的羊毛。**微博推过来的消息会带上原帖的第一张图**（大图，看得清券码和价格），版式是 正文 → 图 → 微博原文链。

**拿博主 UID**：打开博主主页，地址栏形如 `https://weibo.com/u/1234567890`，后面那串数字就是 UID。填进控制台「配置」页的 `WEIBO_UIDS`（逗号分隔）。

**拿 Cookie**（强烈建议）：不填也能跑，但匿名请求大概率被限流、返回空数据。

在控制台「配置」页 → 微博 Cookie 那一行右边，点 **「扫码登录」**。二维码**直接显示在弹窗里**，用微博 App 扫一下，Cookie 自动写回。**零安装**——不用装浏览器，也不用碰 F12。

> 原生扫码调的是微博自己的登录接口。万一哪天微博改了接口导致它失效，控制台会提示你改用**浏览器扫码**兜底，那条路需要装一次 playwright（它会连带下载约 150MB 的浏览器内核，所以没放进核心依赖）：
> ```bash
> pip install playwright

> ```

它判断「登录成功」不是看页面跳没跳，而是**拿刚拿到的 Cookie 去打一次真实接口**，看返回的 `ok` 是不是 `1`——所以扫完就说成功的话，那就是真的能用。

## 九、常见问题

**发 `/w` 没有任何反应**

1. 你是在**私聊**发的吗？群里必须 **@ 机器人**才应答（见[第五节](#五在群里怎么用命令要--机器人)），而且只有「推送到哪些群」里的群才响应。
2. 看「运行」页 NapCat 那一行是不是绿的。不是 → 点「启动」或「扫码登录」。
3. 实时日志里有没有 `WebSocket /onebot/v11/ws [accepted]`？没有 = NapCat 没连上 bot。

**黑框能不能全都不要**

能。`console.bat` 用 `pythonw` 起窗口程序，NapCat 由控制台用无窗口模式启动，两个黑框都没有。NapCat 的日志会混进「运行」页的实时日志里（带 `[NapCat]` 前缀）。

前提是你别再去双击 `napcat.bat`。如果你手动开了一个，控制台的「启动」会先把它停掉。

**停止 NapCat 会不会把我自己的 QQ 也关了**

不会。控制台只杀 NapCat 安装目录底下的进程——虽然它自带的那个进程也叫 `QQ.exe`，和你自己在用的 QQ 重名，但按路径能分得清清楚楚。

**8081 端口被占用**

```bash
netstat -ano | findstr :8081     # 看是哪个 PID 占着
taskkill /PID 那个数字 /F        # 杀掉
```

最常见的原因是启动了两个 bot 实例，它们会抢同一个端口和 NapCat 连接。

**改了配置，但没生效**

- 改「配置」页（`.env`）→ **必须重启**（右上角「重启」）
- 改 `src/plugins/` 或 `src/services/` 下的 `.py` → **必须重启**
- 改订阅 / 品类 / 拦截开关 / 暂停 → 自动热加载，什么都不用做

**机器人被封号了**

QQ 对自动化行为有风控。降低风险：用小号、别在短时间内发大量消息、别加太多群。**风险无法归零，这就是为什么第一条就说别用主号。**

**能不能把控制台或 8081 端口暴露到局域网？**

**不要。** `HOST` 请保持 `127.0.0.1`。8081 端口上除了 NapCat 的反向连接，还有内部接口 `/api/internal/resend`，控制台靠它让 bot 补发消息，它只靠一个存在本机文件里的 token 保护。绑到 `0.0.0.0` = 把「让你的 QQ 号往群里发消息」这个能力交给整个局域网。

**Linux / macOS 上能跑吗？**

**只支持 Windows。** 桌面控制台是彻底的 Windows 程序——`gui/` 下每个文件都在用 `netstat`、`taskkill`、PowerShell 或 tkinter。而控制台是这个项目唯一的操作界面，配置、启停、扫码、订阅全在里面。

bot 本体（`src/` 和 `bot.py`）里确实没有任何 Windows 专属代码，依赖也都是跨平台的，CI 每次都在 Ubuntu 上跑完整测试。所以它**大概**能在 Linux 上跑起来，但作者从没端到端试过，别指望这份文档能帮到你。

## 十一、已知限制

- **AI 语义匹配不是确定性的。** 同一条消息两次判定可能不同。订品牌名（如「八喜」）时偶尔会命中同类的其它品牌。
- **`estimate_paid_price` 是启发式的。** 它优先取 `【N】` 里的数字当到手价；如果那个 `【N】` 其实是赠品价或凑单价，会低估。
- **程序不做乘法。** 只写了单价的帖子读不出总价（见[第三节](#三三类订阅--价格上限)）。反过来，只写了总价的帖子也读不出单价——它不知道一箱有几瓶。
- **`/查` 受轮转限制。** `events.jsonl` 到 ~2MB 会砍掉旧的一半，所以「最近两天」不保证真能翻到 48 小时前。
- **群订阅没有权限控制。** 同群任何人都能删掉别人加的订阅。
- 京东 union-click 超长链无法转短链；微博橱窗 API 不暴露京东直链。

## 十二、开发

```bash
cd tests && python -m unittest discover -v
```

用标准库 `unittest`，**不联网、不需要 NapCat、不碰你的真实数据**（所有数据路径都被重定向到临时目录）。几秒钟跑完。

动 `price_checker.py` 或 `matcher.py` 之前，务必拿 `src/data/events.jsonl` 里的真实历史消息回放一遍，和旧实现逐条 diff——这个仓库里每一次过滤/价格改动都是靠「历史数据零回归」过关的。单元测试保结构，回放保语义。
